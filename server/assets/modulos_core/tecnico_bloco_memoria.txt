## Critérios de Ativação Expandidos

### Registro Obrigatório:
1. Intensidade >= 7
2. Padrão comportamental identificado pela 2ª vez
3. Insight de nível sistema ou essência
4. Decisão significativa tomada
5. Ponto de virada emocional
6. Ativação de protocolo laranja/vermelho
7. Pedido explícito do usuário

### Registro Recomendado:
1. Intensidade >= 5 com novo conteúdo
2. Primeira aparição de tema importante
3. Mudança notável no estilo de comunicação
4. Recurso novo identificado
5. Experimento aceito e realizado

### Flags de Contexto:
- `hasTechBlock`: Incluir análise técnica
- `saveMemory`: Garantir persistência
- `significantInsight`: Detalhar descoberta
- `patternDetected`: Mapear sistema completo
- `turningPoint`: Marcar mudança de fase
- `crisisProtocol`: Documentar intervenção
- `resourceEmergence`: Registrar força descoberta

## Protocolo de Recuperação V2

### Quando Referenciar:
- Padrão reaparece após >= 3 turnos
- Contradição significativa com memória anterior
- Evolução clara de tema registrado
- Pedido de histórico pelo usuário
- Necessidade de evidenciar progresso

### Como Referenciar:
```
"Lembro que [tempo] você mencionou [palavras exatas breves]..."
"Isso conecta com [padrão] que apareceu quando..."
"Diferente de [situação anterior], agora percebo..."
```

### Limites:
- Máximo 2 referências por resposta
- Mínimo 3 turnos entre referências do mesmo registro
- Nunca usar memória para contradizer usuário
- Sempre permitir reinterpretação de eventos passados

## Compressão Inteligente

Para sessões longas (>20 turnos), comprimir registros antigos mantendo:
- Palavras-chave exatas do usuário
- Padrões principais identificados
- Pontos de virada significativos
- Recursos descobertos
- Insights de nível sistema/essência

Descartar:
- Detalhes processuais redundantes
- Hipóteses não confirmadas
- Métricas de baixa relevância
- Notas já integradas em padrões

## Privacidade e Ética

- Anonimizar referencias a terceiros
- Não registrar informações identificáveis desnecessárias
- Marcar registros sensíveis com flag de privacidade
- Permitir pedidos de "esquecimento" de tópicos específicos
- Separar fatos de interpretações nas notas
